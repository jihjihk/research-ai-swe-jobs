    T05: Cross-dataset comparability

    Finding

    The three datasets show statistically significant distributional differences across nearly every dimension tested, but the differences are moderate in practical magnitude and largely attributable to known structural factors rather than measurement artifacts. Description lengths grew ~50% from 2024 to 2026 (median 3,124 -> 3,948 -> 5,110 chars), consistent with real posting inflation. Company overlap is low (Jaccard 0.08-0.12) but top-50 overlap is reasonable (20-32%), reflecting natural employer turnover plus different dataset sizes. Geographic distributions are broadly stable (CA, TX, VA dominate in all three periods), with Cramer's V = 0.14-0.18 indicating small effect sizes. The critical finding is that seniority distributions are nearly identical between arshkon and scraped (chi2 p=0.096, not significant) but sharply different from asaniczka (chi2 p<1e-55), confirming asaniczka's missing entry-level labels as a structural gap rather than a temporal shift.

    Implication for analysis

    RQ1 (junior share): The arshkon-vs-scraped comparison is the only valid two-period seniority comparison. Entry-level share is 4.4% in arshkon (2024-04, n=89) and 5.7% in scraped (2026-03, n=132). Asaniczka cannot contribute to entry-level trend analysis due to its systematic zero entry-level coverage. Any RQ1 analysis must flag the arshkon-only historical baseline and the thin n=89 anchor.

    RQ2 (task migration): Description length inflation (50% growth) means raw keyword counts must be normalized by description length or use rate-per-posting measures. The description_core field amplifies this problem (core_length KS D=0.39 for asaniczka vs scraped), likely because boilerplate removal is inconsistent across periods. Prefer description for keyword work.

    RQ3 (employer-requirement divergence): Company turnover across periods (Jaccard ~0.10) means cross-period requirement comparisons should focus on the ~400-670 overlapping companies or use industry/size strata rather than assuming company-level panel continuity.

    Data quality note

    1. KS tests are all significant (p < 1e-14 for every pairwise comparison on every dimension), but this is expected given sample sizes of 4K-22K. Effect sizes (KS D-statistics, Cramer's V) are the interpretively meaningful quantities.

    2. Industry labels are not comparable across sources. Scraped data uses multi-industry compound labels (e.g., "Accounting, Financial Services, and IT Services and IT Consulting") while arshkon uses single-industry labels. The industry chi-squared (V=0.34) overstates real difference because it conflates label format with industry composition. Industry comparisons require label harmonization before analysis.

    3. Asaniczka seniority caveat. Asaniczka's seniority distribution (97.6% mid-senior among known) is an artifact of having only mid-senior and associate native labels. This is documented in the schema but bears repeating: asaniczka should be excluded from seniority-stratified analyses or used only for mid-senior subgroup comparisons.

    4. Title vocabulary overlap is very low (Jaccard 0.04-0.06). This partly reflects how LinkedIn title strings work (minor phrasing differences create distinct normalized titles). The unique-to-scraped titles show AI-era patterns ("ai data engineer", "ai full stack engineer manager", "go/golang software engineer - ai training") that do not appear in 2024 data.

    Action items

    1. Normalize keyword/pattern counts by description length in RQ2 analysis to avoid confounding real requirement changes with posting length inflation.
    2. Harmonize industry labels before any industry-stratified analysis. Scraped compound labels need splitting or mapping to single-industry categories.
    3. Use arshkon as the sole historical anchor for entry-level analysis. Document the n=89 entry-level SWE baseline prominently in limitations.
    4. Use description (not description_core) for cross-period keyword analysis. The core_length distributions diverge much more than description_length for asaniczka vs scraped (D=0.39 vs D=0.20), suggesting boilerplate removal introduces period-specific artifacts.
    5. Consider restricting cross-period company analyses to the ~380 overlapping companies between arshkon and scraped for within-company temporal comparisons.
