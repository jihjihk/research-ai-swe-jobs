    T01: Column Coverage Audit

    Finding

    Of 76 columns in stage8_final, 38 column-source pairs have >50% null rates, but the damage is concentrated in source-specific metadata fields (search parameters, company companion data, scrape timestamps) rather than core analysis columns. The three critical analysis columns -- is_swe, period, and description -- have 100% coverage across all sources after default filters. The binding constraint is seniority_final: 54.6% unknown for arshkon SWE, 29.3% unknown for asaniczka SWE, and 44.6% unknown for scraped SWE.

    Effectively unusable for cross-period analysis

    These columns exist in only one source and cannot support 2024-vs-2026 comparisons:

    • company_size, company_size_raw, company_size_category, company_id_kaggle (arshkon-only)
    • asaniczka_skills (asaniczka-only)

    • scrape_date, scrape_week (scraped-only)

    Implication for analysis

    1. Cross-period comparisons must rely on text-derived features. Structured fields like skills_raw, company_industry, and company_size are not consistently available. The description text is the universal analysis substrate.
    2. Seniority is the weakest link for RQ1. The unknown rate (29-55% for SWE) means seniority-stratified analyses discard a large fraction of data. LLM-based seniority classification (Stage 12) is the highest-impact improvement.
    3. Entry-level baseline is extremely thin. Only 89 arshkon SWE postings have native entry-level labels. The 124 asaniczka "entry" labels are imputed from title keywords, not platform labels.
    4. Geography is moderately available (~56-75% metro_area coverage), sufficient for geographic controls but not for metro-level deep dives on thin seniority slices.

    Data quality note

    • Coverage rates are computed after default filters (LinkedIn, English, date_flag=ok), which remove ~60K rows from the 1.22M total.
    • "Non-null" for VARCHAR columns means both non-NULL and non-empty-string.
    • The seniority_final column is always populated (100%) but 72.5% of values are "unknown", which is functionally equivalent to null for analysis purposes.

    Action items

    1. Downstream tasks must not assume skills_raw or company_industry are available cross-period. Use description-derived features instead.
    2. Any seniority-stratified analysis must report the effective sample size after excluding unknowns, and discuss the potential bias from non-random missingness.
    3. Stage 12 (LLM seniority) is the single highest-value pipeline improvement for RQ1-RQ3.
    4. For RQ2 keyword/pattern work, prefer description over description_core given the 44% accuracy of boilerplate removal.
