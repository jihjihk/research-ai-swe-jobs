# T02: Seniority Label Comparison

## Finding

The seniority label variants produce **contradictory RQ1 results**: rule-based variants (A, C) show entry-level share *increasing* from 2024 to 2026, while native-label variants (B, E) show it *decreasing*. This discrepancy is driven by a pipeline bug where `native_backfill` effectively does not fire for SWE rows (only 1 of 30,283 SWE rows gets backfilled), leaving 9,490 SWE rows with available `seniority_native` labels unused. The rule-based classifier has excellent mid-senior recall (98%) but poor entry (30-61%) and associate (8-15%) recall, systematically underclassifying junior roles.

## Cross-tabulations

### seniority_imputed vs seniority_final (SWE)
- **100% agreement, kappa = 1.0** -- these columns are identical for SWE rows, confirming native_backfill is not functioning.

### seniority_native vs seniority_final (SWE, both non-null/unknown)
- 18,754 rows evaluable
- **93.3% raw agreement, Cohen's kappa = 0.34 (fair)**
- High raw agreement is inflated by the mid-senior base rate (~92% of evaluable rows). Kappa reveals poor agreement beyond chance.

### seniority_native vs seniority_imputed (SWE, both non-null, imputed non-unknown)
- Identical to native vs final (confirming the bug).

## Native backfill bug

`resolve_seniority_final()` in `stage5_classification.py` (line 624) is designed to backfill from `seniority_native` when the rule-based classifier returns "unknown". However, it only does so when `infer_seniority_family()` returns "swe", "adjacent", or "control". The `infer_seniority_family()` function uses `swe_lookup` through `normalize_swe_title_key()`, while `is_swe` is set via a different path through `title_normalized`. This mismatch causes many rows with `is_swe = true` to have family = "other", bypassing native_backfill.

Impact of the bug on SWE seniority distribution:

| Seniority | Current count | With backfill fix | Multiplier |
|---|---|---|---|
| entry | 345 | 1,273 | 3.7x |
| associate | 290 | 1,623 | 5.6x |
| mid-senior | 18,885 | 26,103 | 1.4x |
| director | 117 | 128 | 1.1x |
| unknown | 10,646 | 1,156 | 0.1x |

Unknown rate drops from 35.2% to 3.8% with the fix.

## RQ1 Junior share by variant

| Variant | 2024-01 | 2024-04 | 2026-03 | Direction |
|---|---|---|---|---|
| A: seniority_final (non-unknown) | 0.81% | 4.42% | 5.65% | INCREASING |
| B: seniority_native only | 0.00%* | 22.91% | 15.49% | DECREASING |
| C: high-confidence (title_keyword + native_backfill) | 0.88% | 5.06% | 6.67% | INCREASING |
| E: hypothetical with backfill fix | 0.57% | 18.69% | 12.28% | DECREASING |

*asaniczka has zero entry-level native labels

Variants A and C show entry share *increasing* over time, suggesting expanding entry-level hiring. Variants B and E (which include native labels) show entry share *decreasing* from 2024-04 to 2026-03, suggesting shrinking entry-level share. **The direction of the RQ1 finding depends entirely on which seniority operationalization is used.**

The discrepancy arises because:
1. The rule-based classifier misses ~70% of entry-level postings (by not recognizing entry-level titles without explicit "junior/intern/entry" keywords).
2. The native label captures entry-level postings that have generic titles like "Software Engineer" but platform-assigned entry seniority.
3. The 2024-04 period (arshkon) has much richer native entry labels than 2026-03 (scraped), creating a measurement artifact.

## Implication for analysis

**RQ1 results are NOT ROBUST to seniority operationalization.** This is a blocking issue. Until the native_backfill bug is fixed AND the LLM seniority classifier runs, RQ1 conclusions about junior-share trends should be considered provisional. The rule-based classifier alone cannot reliably answer RQ1 because it systematically undercounts entry-level postings.

Recommended priority:
1. Fix the native_backfill bug in Stage 5 (align `infer_seniority_family` with `is_swe` classification logic).
2. Run `seniority_llm` at scale to provide a third independent signal.
3. Report RQ1 using multiple variants with explicit uncertainty bounds.

## Data quality note

- asaniczka (2024-01) has zero entry-level native labels, making it impossible to use native-only analysis for the largest period.
- arshkon's 89 native entry-level SWE postings are the sole historical entry-level anchor.
- Cohen's kappa of 0.34 indicates the classifier and native labels have only "fair" agreement beyond what chance would predict, concentrated in non-mid-senior classes.
- Scraped data has higher entry recall (61% vs 30%) than arshkon, suggesting classifier performance varies with data vintage.

## Action items

1. **[BLOCKING] Fix native_backfill bug in Stage 5.** The `infer_seniority_family()` function must align with the `is_swe` classification path so that SWE rows get backfilled. This alone would reduce SWE unknown rate from 35% to 4%.
2. **[HIGH] Run seniority_llm.** The LLM classifier is the best path to resolving the entry/associate undercount without relying solely on platform native labels.
3. **[HIGH] Report RQ1 with variant sensitivity table.** Any RQ1 finding must show results under at least variants A and E (or the backfill-fixed equivalent).
4. **[MEDIUM] Investigate why scraped has 2x better entry recall than arshkon.** This could indicate title wording changes between 2024 and 2026, or it could indicate selection bias in the scraper's query design.
5. **[LOW] Consider if associate/director misclassification matters for RQ1.** RQ1 focuses on junior share, and associate recall (~10%) is poor but associate-to-mid-senior confusion may be less consequential if the 3-level collapse (junior/mid/senior) is used.
