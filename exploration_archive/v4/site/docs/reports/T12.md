# T12 — Open-ended Text Evolution

**Date:** 2026-04-09
**Scope:** SWE LinkedIn only (`is_swe=true`, `source_platform='linkedin'`, `is_english=true`, `date_flag='ok'`), n=63,294 (2024 = 28,232; 2026 = 35,062).
**Text source:** Raw un-stopworded text via `COALESCE(description_core_llm, description_core, description)` from `data/unified.parquet` (same as T13). Markdown escapes (`C\+\+` -> `C++`, `.NET\-Core` -> `.NET-Core`) un-escaped in the tokenizer — see Method.
**Primary seniority signal:** combined best-available column; validated with YOE proxy.
**Section isolation:** per-posting section text extracted via T13's `section_classifier.extract_sections()`. "Core sections" = summary + responsibilities + requirements + preferred.

## Bottom line

1. **AI/LLM vocabulary is the dominant content shift.** `agentic` (445x), `rag` (76x), `claude` (350x), `cursor` (604x), `langgraph` (20,000x, zero baseline), `llm` (15x accelerating), `genai` (32x), `prompt` (16x) all emerge or accelerate dramatically between 2024 and 2026. This is consistent whether you look at full text or section-filtered core content, unigrams or bigrams. **The AI shift is genuine content evolution, not boilerplate padding.**
2. **Boilerplate explosion is real but separable.** 6 of the top-100 full-text 2026-favored terms are boilerplate tokens (`accommodation, reasonable, disabilities, seekers, notice, employees`). Thirty of the top-100 terms are *unique to the full-text view* and vanish when you restrict to core sections — most of those are pay-transparency / legal / benefits language. T12 recommends reporting core-filtered term lists as primary, full-text as supplementary.
3. **Tech-stack shift is modest in doc-frequency, large in per-token rate.** C++ still appears in 24% of 2026 postings (up from 14.75% in 2024 by document frequency), but its per-token rate dropped because postings got longer — Fightin' Words flags C++ as "2024-favored" (z=-11.96). **Important caveat for the paper: a term can be "declining" by rate while still appearing in more postings.** We report both metrics.
4. **"Responsibilities" verbs shift toward AI-native idioms.** Mid-senior comparisons (core sections): 2026 gains `workflows, observability, agentic, influence, outcomes, end-to-end, cloud-native, tooling, translate, pipelines, high-impact`. 2024 had `qualifications, required, responsibilities, powerful, cloud-based, trends` — literally less specific verbiage.
5. **Entry-level corpora pass the relabeling test.** Entry 2026 vs mid-senior 2024 top distinguishing terms are *entry-specific* (`pursuing, intern, summer, enrolled, currently, internships, coursework, graduation, undergraduate`), not *senior-rebranded*. The 2026 entry pool is not a relabeling of 2024 seniors — they're genuinely describing different roles.
6. **BERTopic corroborates the AI story.** The largest-growing BERTopic topic is `0_ai_learning_machine learning_ml` (3.2% -> 15.2%, +11.95pp). Second largest is `4_team_work_design_technical` (+5.0pp). Biggest shrinkers: infrastructure/devops (-0.4pp), network/server admin (-1.4pp), embedded/firmware (-1.3pp). **The "scope inflation" narrative is really an "AI/LLM injection plus org-scope rhetoric" narrative.**

## Method

### Text preparation

- Loaded cleaned-text metadata from `exploration/artifacts/shared/swe_cleaned_text.parquet`, then joined the raw un-stopworded text from `data/unified.parquet` (same COALESCE order used by the shared cleaned text build). Company tokens from `company_stoplist.txt` stripped at tokenization time, plus English stopwords and a small hand-curated filler list.
- **Markdown escape fix (critical):** Scraped LinkedIn text contains markdown escapes for regex special characters (`C\+\+`, `F\#`, `\*` for bullets, `3\.` for numbered lists, hyphens as `\-`). The initial T12 run produced the spurious finding that C++ had disappeared entirely from 2026 postings (0/35,062). Direct DuckDB checks found the literal string `C\+\+` in 8,379 of those postings (23.9%). The tokenizer now pre-processes text by removing backslash escapes for `*+#.-!()[]_`. This is **the single most important technical lesson** from this task; any downstream text analysis on scraped data must handle these escapes.
- Tokenization: `[A-Za-z][A-Za-z0-9+#\.\-]{1,30}` (case-insensitive, allows `+`, `#`, `.`, `-` so `c++`, `c#`, `node.js`, `cloud-native` survive as single tokens). Document-level deduplication (one count per posting) is used for all term frequencies.
- Stoplist union: 10,652 company-name tokens + sklearn English stopwords + ~45 hand-curated filler words.

### Fightin' Words (Monroe et al. 2008)

- Log-odds ratio with informative Dirichlet prior. Prior = pooled term counts from both corpora (halved). For each term, z = (log_odds_26 - log_odds_24) / sqrt(1/num_26 + 1/num_24). Positive z favors 2026; negative favors 2024.
- Minimum doc-frequency filter: 20 postings overall for unigrams, 15 for bigrams.
- Runs: (a) all 2024 (arshkon + asaniczka) vs scraped 2026 on full text; (b) same on core sections only; (c) same with bigrams in both variants.

### Document-frequency rate metrics

- `emerging`: df_2024 < 0.1% AND df_2026 >= 1.0%
- `accelerating`: df_2024 >= 0.1% AND df_2026 >= 1.0% AND ratio (df_2026/df_2024) >= 3.0
- `disappearing`: df_2024 >= 1.0% AND df_2026 < 0.1%

### BERTopic

- Precomputed shared embeddings (all-MiniLM-L6-v2, 384-dim, from `exploration/artifacts/shared/swe_embeddings.npy`).
- Tuned UMAP (n_neighbors=15, n_components=5) and HDBSCAN (min_cluster_size=50) after the BERTopic default produced a single mega-topic.
- Stratified sample of ~8,000 postings (4,000 per period) from the core-text field. Topic-per-period shares computed via crosstab.

### Sanity check: C++ did NOT disappear

Initial T12 run flagged c++ as "disappearing" with a 14.75% -> 0.0% document frequency. Investigation revealed this was a tokenizer bug: scraped text contains markdown-escaped `C\+\+`. After the fix, c++ document frequency is actually higher in 2026: **23.9% (count 6,258) vs 14.8% (count 4,165)**. But its per-token rate *dropped* (1.80 -> 1.26 per 1k) because 2026 postings are longer. Fightin' Words (which normalizes per token) correctly labels it as "2024-favored" by rate (z=-11.96), even though more postings mention it. **This is a methodological lesson: Fightin' Words z-scores are about relative per-token rate, not about "which corpus has more postings mentioning a term".** Both metrics are reported in all per-term tables.

## Surprises & unexpected patterns

1. **The C++ tokenization bug is not just a local issue — and it HAS propagated to the shared tech matrix.** The initial T12 run would have reported "C++ disappeared from SWE postings" — a false finding driven entirely by markdown escaping in scraper output (`C\+\+`). Every downstream text analysis on scraped data needs the unescape step.
   **CRITICAL: `exploration/artifacts/shared/swe_tech_matrix.parquet` is broken for C++ and C#.** Verified by direct DuckDB LIKE matching: C++ appears in 19.2% of scraped SWE postings (literal `C\+\+`), C# in 11.3% (literal `C#`). The tech matrix reports c_cpp at 0.5% in scraped (185 rows) and csharp at 0.15% (54 rows). Two root causes: (a) the `\bc\+\+\b` regex pattern is fundamentally broken — `\b` doesn't match between `+` and end-of-word; (b) the matcher runs on `description_cleaned` which preserves markdown backslash-escapes. The tech matrix was used to compute Wave 1's tech-count calibration ratios (T05) and any tech-prevalence claim in T04/T05/T06 that involves c_cpp, csharp, or other `+`/`#` tokens is unreliable. **Recommended fix:** rebuild the tech matrix with (a) unescaped text, (b) regex patterns without `\b` boundaries on non-word chars (use `(?:^|\W)c\+\+(?:\W|$)` or simple substring matching). See `T12_text_evolution.py` `_MD_ESCAPE_RE` and tokenizer for reference.
2. **Amazon's share of scraped postings is large enough to skew top-term lists.** Bigrams like `destination stage`, `stage careers`, `amazon.jobs how-we-hire`, `empowers amazonians`, `amazonians deliver` make the top-30. A company-capped version of T12 (cap each company at N postings before running Fightin' Words) is recommended for final analysis.
3. **`cursor` (the AI IDE) is now in 2.2% of 2026 SWE postings (count=775).** This is named-product-level adoption of a consumer AI tool in job requirements. In 2024: 5 postings total.
4. **`langgraph` went from 0 to 2.04% (count=719).** A framework released in 2024 is now explicitly required in ~1 in 50 SWE postings.
5. **`mcp` (Model Context Protocol) appears in 2.07% of 2026 postings.** MCP was an Anthropic protocol announcement late 2024; its appearance at >2% means it has crossed from hypothesis into hiring-managerspeak within roughly a year.
6. **Credential vocabulary receding.** `bachelor` (9.14 -> 5.73 per 1k in mid-senior core), `degree` (core equivalent), `qualifications` (3.03 -> 0.25 per 1k in mid-senior core), `required` (5.00 -> 2.23 per 1k in mid-senior core). This is a *stronger* finding than the AI addition — the traditional "hard requirement" vocabulary is being actively stripped out of mid-senior postings. Candidate explanation: market tightening + employer templates dropping strict gatekeeping language.
7. **Organizational-scope vocabulary diversifies.** `influence, outcomes, end-to-end, high-impact, contribute, ownership, governance, long-term, translate, excellence, impactful, thrives` all appear in the core-only top-100. The mid-senior 2026 role is being described more in terms of *what its impact should be* than in terms of *what skills are required*. This is a subtle but consistent shift; combined with the drop in "required/qualifications" it suggests a rhetorical move from "checklist role" to "impact role".
8. **Asaniczka top-boilerplate is clearance vocabulary (defense hiring).** Not a surprise per se, but it means aggregate 2024 stats on "soft-skill share" or "clearance share" are dominated by asaniczka sector composition, not by job-market trends.
9. **Entry-2024 finance contamination.** The arshkon entry-level pool disproportionately contains finance/banking/compliance postings (laundering, vba, macros, econometric, liabilities). This narrowly confirms Wave 1's finding that arshkon native-entry labels are contaminated, but with a new mechanism: it's not that arshkon mislabels seniors as juniors — it's that arshkon's entry pool is skewed toward an industry vertical whose "entry" postings are heavy on finance vocabulary.
10. **BERTopic found an AI topic that is genuinely a single coherent cluster** — the `agentic/llm/rag/prompt` cluster hangs together as one topic, not split across 5. This is evidence that "generative AI" is crystallizing as a recognized sub-field within SWE hiring, not just a buzzword sprinkle across existing topics.

## Essential sensitivities

- **(a) Aggregator exclusion:** Not applied. Should be rerun for the sanity check.
- **(b) Company capping:** Not applied in this pass. Amazon (and a few other large employers) can individually skew top-term lists — see the `destination stage` / `amazon.jobs` bigrams. **Recommended next step:** cap each canonical company at 200 postings and rerun; the AI-core findings should survive since they're distributed across many companies, but the bigram list will lose Amazon-specific boilerplate.
- **(d) Description text source:** Uses raw COALESCE-based text; mixes LLM-extracted core (high coverage in kaggle) with rule-extracted core (high coverage in scraped). See T13 caveat — the section split is mildly biased by this, but the unigram-level term-rate changes should be robust since both text sources preserve content words.
- **(e) Source restriction:** LinkedIn only; Indeed excluded.
