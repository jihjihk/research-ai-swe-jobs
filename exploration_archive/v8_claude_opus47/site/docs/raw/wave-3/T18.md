# T18 — Cross-occupation boundary analysis

Agent K, Wave 3. 2026-04-17. Input: `data/unified.parquet`.

## Headline (<200 words)

The SWE-specific restructuring framing is **defensible for the AI, requirements, and tech-stack metrics** but **collapses to a macro template shift for description length and soft skills**. DiD (SWE change minus control change) is ~99% of SWE-only change for AI-mention (strict), AI-mention (any), `has_ai_tech`, `tech_count`, and `ai_tech_count`; ~70% for `org_scope_count` and `requirement_breadth`. `desc_len_chars` DiD is only 37% of SWE-only (mostly macro); `soft_skill_count` DiD is ~0 (fully macro). The **requirements-section shrink is NOT universal**: SWE -10.7 pp, adjacent -10.9 pp, control **+0.9 pp**. Control expanded requirements-section share while SWE compressed it — the two groups move in opposite directions. The AI gradient is **widening**: SWE(strict) went 0.010 → 0.153 (+0.142); adjacent 0.007 → 0.118 (+0.111); control 0.002 → 0.002 (≈0). Control is not catching up. Among adjacent titles, ML engineer (+0.54) and data scientist (+0.15) overtake SWE (+0.13) on AI-mention — SWE is no longer the frontier; it is being surrounded by AI-native roles. Sensitivities (aggregators, tier) confirm direction. Boundary TF-IDF cosine SWE↔adj drops 0.80 → 0.75 (slight divergence, not blur).

### Interpretation

- **AI content is SWE-specific.** Nearly all of the SWE AI-mention growth is over and above what control white-collar did. Control moves 0.000–0.003 in absolute prevalence.
- **Breadth and scope are SWE-specific.** Control's requirement_breadth rose only 0.96 compared to SWE's 3.41; 72% is SWE-specific.
- **Length and soft-skill growth are macro.** Control postings got 725 chars longer and gained ~0.48 soft-skill tokens — that is a universal "postings got more verbose" shift. Only the ~423-char SWE excess is plausibly SWE-specific.
- **Education-level change is small and not consistent.** SWE dropped slightly while control rose; the DiD flag is driven by a tiny denominator.

## Surprises & unexpected patterns

1. **Control postings do not adopt AI-mention at all.** Raw prevalence 0.002 → 0.002. I expected some white-collar drift from HR/marketing/sales writing "AI-driven" in descriptions; it hasn't materialized in LinkedIn non-SWE, non-adjacent postings at the strict pattern level. Under broad pattern control goes 0.011 → 0.078 (+6.7 pp), so there is a real but small macro shift — but strict pattern is clean.
2. **Requirements-section behaves in OPPOSITE directions for SWE vs control.** SWE contracts (-10.7 pp) while control expands (+0.9 pp). This is much stronger evidence for a SWE-specific template restructuring than simply "everyone compressed requirements."
3. **ML Engineer and Data Scientist overtake SWE on AI-mention.** ML engineer post-period ai_strict is 0.54 (SWE: 0.14); data scientist is 0.19. Post ai_strict gap SWE minus ML-eng becomes -0.40 — the AI-native adjacent roles are now AHEAD of general SWE. The "AI frontier" inside LinkedIn postings has moved from SWE to ML-eng/DS.
4. **TF-IDF cosine decreases** between SWE and adjacent (0.80 → 0.75). Content-level divergence coexists with AI-mention convergence. Adjacent roles are adopting similar AI vocabulary but their tech stacks and duties remain distinct — "vocabulary overlap, content divergence."
5. **Education level drops in SWE but rises in control** (SWE -0.06 vs control +0.12). SWE postings in 2026 mention degrees less often; control postings mention them more. Possible signal: credentialing compression on SWE side (AI skills > formal education), ad-hoc degree gating on the control side.
6. **`soft_skill_count` moves identically across groups.** SWE +0.48, control +0.48. Soft-skill inflation is the purest macro / template pattern in the data. Useful null control for the AI/tech findings.
7. **Aggregator exclusion INCREASES the SWE-specific signal** on breadth/length (DiD 2.45 → 2.86). Aggregator postings are more generic and their inclusion pulls SWE means toward the middle. The headline finding is robust — aggregators weakly *attenuate* rather than create the result.

## Data-quality caveats

- **arshkon window (2024-04-05 to 2024-04-20) contains within-window heterogeneity** in AI-mention (range 0.00–0.10 across days, CV 1.44). Treat arshkon as a 2-week sample, not a point. Reported in T19 Step 2.
- **Tech-count on RAW description may over-count** for long control/adjacent postings containing "python" mentions in unrelated contexts. Sensitivity (a) `no_aggregators` still preserves direction, so not a fatal confound.
- **Boundary TF-IDF uses raw text** and company-stoplist stripping; N=200 per cell. For formal work, scale to 1,000+ per cell and use description_core_llm on LLM-labeled subset only with coverage caveat.
- **Section classifier precision** is not exhaustively validated on adjacent/control. If adjacent/control postings use different section-header conventions, the absolute shares may shift; the *change* per group is less vulnerable because it's a within-group comparison.
