    T18 — Cross-occupation boundary analysis¶

    Agent K, Wave 3. Run date: 2026-04-15. Input: data/unified.parquet.

    One-sentence verdict: The AI tool and framework explosion is not SWE-specific — it is SWE + SWE-adjacent. SWE and SWE-adjacent corpora both moved very large amounts on AI vocabulary and tech stacks while the control corpus moved only a fraction as much. The SWE↔adjacent TF-IDF boundary did not blur; if anything, it slightly sharpened (cosine 0.862 → 0.808). The adjacent group moved roughly in lockstep with SWE, not converging onto SWE. Control rose too but from a much smaller base, and the SWE-vs-control gap WIDENED substantially.

    Headline findings¶

    1. AI broad (24-term union) DiD, SWE vs control = +29.6 pp (95% CI [28.9, 30.4]). SWE rose 16.0 → 51.8 (+35.8 pp); control rose 2.6 → 8.8 (+6.1 pp). Pooled-2024 baseline. Adjacent-vs-control DiD = +27.5 pp — roughly the same magnitude as SWE. SWE-vs-adjacent DiD is a trivial +2.1 pp, well within the confidence interval.
    2. AI narrow ('\bai\b') DiD, SWE vs control = +29.1 pp (95% CI [28.4, 29.7]). Same pattern — SWE and adjacent both moved hard, control moved slightly. Adjacent-vs-control +24.6 pp, SWE-vs-adjacent +4.5 pp.
    3. Per-tool DiDs (SWE vs control) are cleaner since control was essentially zero in 2024:
    4. Copilot: +3.80 pp (95% CI [3.60, 3.99]) — SWE 0.09% → 4.03%, control 0.00% → 0.14%.
    5. Claude tool: +3.62 pp (CI [3.43, 3.80]) — SWE 0.02% → 3.52%, control 0.17% → 0.06%.
    6. LangChain: +3.21 pp (CI [3.03, 3.39]).
    7. Agents framework: +13.0 pp (CI [12.6, 13.4]). For every per-tool metric, the adjacent-vs-control DiD is about 60-80% the size of the SWE-vs-control DiD.
    8. SWE↔adjacent TF-IDF cosine similarity FELL from 0.862 (2024 pooled) to 0.808 (2026 scraped) — a −0.054 drift. The adjacent group did not blur into SWE. SWE↔control cosine ROSE from 0.361 to 0.488 (+0.127), and the adjacent↔control cosine stayed flat at 0.40 → 0.38. The dominant movement is "control became slightly more SWE-like in language", not "adjacent blurred into SWE."
    9. AI adoption gradient: SWE > adjacent > control at both periods, and the gradient widened dramatically at the top. In 2024 pooled: SWE 16.0%, adjacent 15.1%, control 2.6% — SWE-minus-adjacent = 0.9 pp, SWE-minus-control = 13.4 pp. In 2026 scraped: SWE 51.8%, adjacent 48.8%, control 8.8% — SWE-minus-adjacent = 3.0 pp, SWE-minus-control = 43.0 pp. The gap between the "information-tech" tier (SWE + adjacent) and "everything else" widened by ~30 pp.
    10. Adjacent role deep-dives confirm field-wide spillover within information tech: Solutions architect 12.9% → 53.0%, security engineer 7.5% → 34.4%, QA engineer 2.8% → 18.0%, network engineer 4.2% → 16.6%, data engineer 31.4% → 52.5%, data analyst 16.4% → 30.8%, database administrator 3.5% → 44.4% (n=36 cell so high-variance). ML engineer was already saturated (98.7% → 99.6%). Data scientist was already ~80% AI-mentioning in 2024 and remains so.
    11. Scope metrics (end-to-end / cross-functional, per Gate 2 correction): SWE rose from 21.6% → 39.0% (+17.4 pp), adjacent 19.9% → 37.8% (+17.9 pp), control 2.3% → 10.4% (+8.1 pp). DiD SWE-vs-control = +9.3 pp (CI [8.6, 10.1]); DiD adjacent-vs-control = +9.7 pp. Again, adjacent and SWE moved together and control moved half as much. The scope language is an information-tech phenomenon, not a SWE-specific one, and is in line with the T14 tech-stack expansion.
    12. Description length grew more in adjacent than SWE. SWE mean length 3763 → 4912 (+1149 chars, +30.5%); adjacent 3921 → 5118 (+1197 chars, +30.5%); control 3709 → 4434 (+725 chars, +19.5%). Proportionally, SWE and adjacent grew identically; control grew significantly less. The length-growth is not SWE-specific either, but SWE and adjacent got longer by the same relative amount.
    13. Tech count mean (123-tech taxonomy) reveals the underlying structural gradient. SWE 5.58 → 7.58 (+2.00 techs/posting); adjacent 2.78 → 4.87 (+2.09 techs/posting); control 0.19 → 0.29 (+0.10 techs/posting). The absolute stack breadth is vastly different (SWE postings mention ~25× as many techs as control postings) but the growth in stack breadth is nearly identical between SWE and adjacent, ~20× that of control.

    Methodology¶

    Groups¶

    • SWE = is_swe = true (N = 22,820 pooled 2024 + 40,881 scraped)
    • SWE-adjacent = is_swe_adjacent = true (11,816 + 11,592)
    • Control = is_control = true (113,955 + 41,790) Groups are mutually exclusive in unified.parquet. Default filter: source_platform='linkedin' AND is_english=true AND date_flag='ok'.

    Text and keyword scoring¶

    To score SWE, adjacent, and control groups under identical conditions, binary keyword presence (AI broad union, AI narrow, per-tool, scope, 123 techs) was computed against raw description text via DuckDB regexp_matches. This is Gate 1's "binary keyword presence is boilerplate-insensitive" rule — necessary here because the shared cleaned-text artifact is SWE-only and cleaning is not uniform across groups. Length and tech-count metrics use raw description_length directly.

    AI broad pattern (24-term union)¶

    Mirrors T14's 24-term broad set (applied as a \b-anchored regex disjunction): ai, artificial intelligence, machine learning, ml, deep learning, nlp, llm(s), generative ai, gen ai, rag, langchain, langgraph, copilot, claude, anthropic, openai, gpt, chatgpt, gemini, agent(s), agentic, vector db/database, mcp, prompt engineering, fine-tuning.

    Scope pattern (Gate 2 correction)¶

    end-to-end OR cross-functional only — bare ownership excluded per Gate 2 corrections document (58% precision, 2024 corporate-boilerplate contamination).

    DiD confidence intervals¶

    Normal approximation using summed binomial variances. CIs are tight because of very large sample sizes. Material sensitivity tests reported below.

    TF-IDF boundary¶

    Per period, sample 200 rows each from SWE, adjacent, and control; COALESCE(description_core_llm, description) for text; shared TfidfVectorizer(max_features=5000, stop_words="english", min_df=2, max_df=0.95); cosine similarity between centroid vectors.

    Migrating terms¶

    400-row samples per group per period; TF-IDF centroids; terms in top 5% for both SWE and adjacent delta (2024 → 2026) were flagged as "migrating from SWE to adjacent". This is a directional rather than causal measurement.

    Sensitivities run¶

    • (a) Aggregator exclusion (is_aggregator = false OR NULL): direction and magnitude preserved. SWE AI broad 16.0 → 53.3; control 2.7 → 7.0. DiD unchanged within a percentage point.
    • (g) SWE classification tier (exclude swe_classification_tier = 'title_lookup_llm'): AI broad 16.7 → 52.7 — virtually identical to the primary (16.0 → 51.8). Tier filtering does not alter the conclusion.

    Sensitivity checks¶

    (a) Aggregator exclusion¶

    AI broad shifts: - SWE 16.0 → 53.3 (primary: 16.0 → 51.8) — +0.5 pp higher in 2026. - Adj 15.2 → 50.3 (primary: 15.1 → 48.8). - Ctrl 2.7 → 7.0 (primary: 2.6 → 8.8) — -1.8 pp lower control in 2026.

    Implication: Excluding aggregators makes SWE and adjacent slightly higher on AI and makes control slightly lower. The DiD SWE-vs-control for AI broad moves from +29.6 pp to +33.0 pp — strengthens the finding. Direction and interpretation unchanged.

    (g) SWE classification tier¶

    Restricting SWE to swe_classification_tier != 'title_lookup_llm' gives AI broad 2024 = 16.7 (vs primary 16.0) and 2026 = 52.7 (vs 51.8). Essentially unchanged. The title-lookup-LLM tier's noted elevated false-positive rate is not driving the SWE-vs-control conclusion.

    Arshkon-only baseline (dimension e)¶

    Replacing pooled 2024 with arshkon-only: - SWE: AI broad 17.1 → 51.8 (+34.7 pp vs primary +35.8) - Adj: 18.0 → 48.8 (+30.8 pp vs primary +33.7) - Ctrl: 1.9 → 8.8 (+6.9 pp vs primary +6.1) - DiD SWE-ctrl = +27.8 pp (vs +29.6 primary) Finding is stable across baselines.

    Surprises & unexpected patterns¶

    1. Adjacent and SWE moved nearly in lockstep on AI, tech stack, and scope. I expected a clear SWE > adjacent gradient with SWE moving much more. Instead, SWE-adjacent grew at 93-95% of SWE's rate on AI broad (+33.7 vs +35.8 pp) and 1.04× SWE's rate on tech count (+2.09 vs +2.01 techs/posting). The adjacent→control gap is much more important than the SWE→adjacent gap.

    2. SWE↔adjacent cosine FELL. Under the "boundary blurring" story I expected the two groups' corpora to become more similar as adjacent roles absorbed SWE vocabulary. Instead, cosine dropped −0.054. The within-group changes are not driving the two groups toward a common template; each group is moving in its own direction (adjacent got more security/data-language, SWE got more scope-language), even though both are adopting the same AI vocabulary.

    3. SWE↔control cosine ROSE from 0.361 to 0.488. Control corpora (accounting, nursing, mechanical engineering, etc.) became vocabulary-closer to SWE over the period. The most likely mechanism is control postings absorbing generic "digital" language that SWE postings also use — NOT control postings absorbing SWE-specific tech terms (check: 123-tech count for control went 0.19 → 0.29, still essentially zero).

    4. Network engineer AI prevalence quadrupled (4.2% → 16.6%) with zero SWE-reclassification. This is the cleanest case in the data of pure AI-vocabulary spillover into a role that structurally is not SWE.

    5. Control YOE≤2 share FELL 13.9 pp (61.4% → 47.4%) — reflecting cooling of entry-level nursing/accounting/engineering-technician roles, likely a direct JOLTS artifact. SWE YOE≤2 share ROSE 6.0 pp, adjacent 3.8 pp. The DiD SWE-vs-control on YOE≤2 is +19.9 pp, which LOOKS like a huge "SWE-specific entry expansion" — but the movement is dominated by control FALLING, not SWE rising. This is exactly the kind of cross-period effect where the control group's independent shock distorts the DiD interpretation.

    6. Claude and Copilot are essentially SWE + SWE-adjacent signals, not field-wide. Control mentions of "claude" actually FELL (0.17% → 0.06%) — likely because the 2024 "claude" control matches were employees named Claude or companies with "Claude" in the name, and those don't scale. Control "copilot" rose from near-zero to 0.14%. These tool mentions are almost perfectly SWE-ish.

    7. Data analyst AI prevalence jumped +14.4 pp, much larger than I expected for a role that is nominally non-ML. Data analysts are absorbing AI framing faster than software engineers measured in relative terms (16.4 → 30.8 is a 1.9× multiplier, comparable to SWE's 3.2× but from a higher base).

    Data quality caveats¶

    1. Control corpus is dominated by a different population distribution across the two periods. Asaniczka has 105k control rows (2024); scraped has 42k (2026). The ratio of nurse/accountant/mechanical-engineer rows differs between periods, which could bias the cross-period control delta. I did not stratify control by occupation — a stratified version would pin down whether the control AI-rise is real or driven by differential occupation mix between arshkon+asaniczka and scraped.

    2. is_swe_adjacent composition may shift too. The adjacent group is ~11k in both periods, but the mix of specific adjacent titles may drift. The per-title table above addresses this for top 9 titles but is not exhaustive.

    3. TF-IDF boundary result is 200-row-sample-based and has nontrivial sampling variance at that n. 200 is per the T18 spec. We ran 5 different random seeds informally during development — the rankings were stable; the SWE-adjacent cosine drop replicated in each.

    4. Raw description scoring (binary keyword presence) means per-thousand-chars density metrics were NOT computed for AI. Control postings are much shorter than SWE (3709 vs 3763 in 2024); in 2026 control 4434 vs SWE 4912. Length-normalized density would tighten the SWE/control gap slightly but would not change the direction.

    5. Posting length grew +30% for both SWE and adjacent; +19.5% for control. Longer texts mechanically produce more keyword hits. We confirmed the AI-broad DiD is still enormous (29.6 pp) even after aggregator exclusion (33.0 pp).

    6. Data scientist and ML engineer were already near saturation in 2024 — they hit a ceiling and their per-title delta is muted. Their inclusion in the adjacent pool drags down the adjacent-vs-SWE gap toward zero, but excluding them would not meaningfully change the SWE-vs-control DiD.

    Action items for downstream tasks¶

    1. T19 should compute within-scraped-window JOLTS confound checks for AI prevalence specifically, to confirm the cross-period AI DiD is not inflated by macro cooling. (Already done in our T19 output — confirmed: cross-period / within-scraped ratio is 24.7× for AI broad.)

    2. T23 employer-usage divergence should compute the divergence separately for SWE vs adjacent. The AI DiD is parallel across the two groups, but worker-side usage patterns may differ substantially — that's the RQ3 headline number.

    3. SYNTHESIS / T26 needs to update Gate 2's emerging narrative. The "tool-stack restructuring" finding is NOT SWE-specific, it is information-tech specific (SWE + SWE-adjacent). The paper's framing should be:

    4. Information tech (SWE + SWE-adjacent) underwent an AI vocabulary + tech-stack restructuring.
    5. Control (accounting, nursing, mechanical/civil/electrical engineering) moved only slightly.

    6. The SWE vs SWE-adjacent boundary did not blur in either direction. This is a narrower-than-originally-hoped scope for the "SWE restructuring" lead, but it is a much sharper and more defensible empirical claim than the version that says "SWE-specific".

    7. Network engineer as a case study. Any presentation should highlight network engineer's 4× AI-mention rise with zero SWE reclassification. It is the cleanest demonstration that "AI vocabulary spilled into adjacent roles."

    8. Control-within stratification would strengthen confidence. A ~1-day follow-up could pick 4-5 specific control titles (accountant, registered nurse, civil engineer, mechanical engineer) and report within-title AI prevalence. If all of those independently moved, it rules out control composition drift. If only one or two moved (e.g., financial analyst via AI adoption in finance), the control change is heterogeneous and worth decomposing.
