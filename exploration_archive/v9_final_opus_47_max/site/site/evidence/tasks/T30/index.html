    T30. Seniority Definition Ablation Panel¶

    Run date: 2026-04-20 Owner: Agent B, Wave 1 Input: data/unified.parquet Sample: is_swe = True under default filter (source_platform='linkedin' AND is_english=true AND date_flag='ok'). Script: exploration/scripts/T30_seniority_definition_panel.py Canonical artifact: exploration/artifacts/shared/seniority_definition_panel.csv (78 rows, 14 columns). Tables: exploration/tables/T30/

    Headline¶

    Direction is robustly consistent across 12 of 13 operationalizations. Every junior definition moves up 2024→2026; every senior definition except S2 (director-only, noisy) moves down. The magnitude varies by family: label-based (J1/J2/S1) says junior share rose ~1.8 pp and senior share fell ~15.7 pp; YOE-based LLM primary (J3/S4) says junior share rose ~5.0 pp and senior share fell ~7.6 pp.

    Primary recommendation (unchanged from preamble): J3 (yoe_min_years_llm ≤ 2) and S4 (yoe_min_years_llm ≥ 5) as primaries, label-based (J1, S1) and title-keyword (J5, S3) as sensitivities. Both primaries are well-powered (MDE = 0.8 pp pooled-vs-scraped; actual effect = 5.0 and 7.6 pp). However, the senior-side asaniczka asymmetry persists under the LLM YOE primary (S4 arshkon 65.0% vs asaniczka 72.1%; within-2024 = −7.1 pp, cross-period = −7.6 pp). S1/S4 cross-period magnitudes are comparable to within-2024 source-composition noise and should be claimed with a "magnitude is sensitive to 2024 source mix" caveat. The direction is still robust; the magnitude is not.

    1. Panel overview¶

    1.1 Shape and denominator summary¶

    The canonical panel has 78 rows (13 definitions × 6 analysis groups). Columns:

    • definition, side (junior/senior), family (label, title_keyword, yoe_llm, yoe_rule)
    • period ∈ {2024-01, 2024-04, pooled-2024, 2026-03, 2026-04, pooled-2026}
    • source ∈ {kaggle_asaniczka, kaggle_arshkon, kaggle_asaniczka+arshkon, scraped}
    • n_of_all, n_of_denominator (integers)
    • share_of_all, share_of_denominator (floats in [0, 1])
    • mde_arshkon_vs_scraped, mde_pooled_vs_scraped (80% power, α=0.05, baseline-dependent)
    • within_2024_effect = arshkon_share − asaniczka_share (calibration noise floor for this definition; nonnull on pooled+scraped rows)
    • cross_period_effect = pooled-2026 − pooled-2024 (nonnull on pooled rows)
    • direction ∈ {up, down, flat} (nonnull only on pooled-2026 rows; flat if |effect|≤0.005)

    Sample of 20 rows where |diff| ≥ 3: reviewing audit_llm_vs_rule_yoe_sample_diff3plus.csv. The dominant pattern is the LLM extracting a higher YOE than the rule, typically when the title carries senior/staff/principal language and the description includes a "5+ years" minimum that the rule captured but the LLM aggregated with other higher signals. Examples:

    • "Senior Software Engineer, Android Platform" — LLM=5, rule=2 (rule caught a weak "2 years" mention elsewhere; LLM correctly weighted the overall role ask).
    • "Senior Java Software Engineer" — LLM=8, rule=5.
    • "Senior Scala Engineer" — LLM=6, rule=3.

    In 2/20 rows the LLM was lower than the rule:

    • "OKTA Application Engineer with Min 5+yrs exp" — LLM=5 (the stated min), rule=10 (pulled a higher number from elsewhere).
    • "Software Engineer 3 (IDN - 051)" — LLM=5, rule=9.

    Net verdict: LLM YOE is the better primary for senior-side ≥ N cutoffs (it correctly raises the floor on senior titles); rule YOE is an acceptable audit. For junior-side ≤ N cutoffs, both agree strongly on the floor (99% of entry-title rows have YOE = 0 in both extractors).

    S4 yoe_llm kaggle_asaniczka 14,765 10,648 72.12%
    S5 yoe_llm kaggle_arshkon 3,454 813 23.54%
    S5 yoe_llm kaggle_asaniczka 14,765 4,062 27.51%
    S4_rule yoe_rule kaggle_arshkon 2,911 1,751 60.15%
    S4_rule yoe_rule kaggle_asaniczka 12,834 8,528 66.45%

    Findings.

    • Label asymmetry is severe: S1 arshkon 43% vs asaniczka 64% = 21 pp gap. Plausible explanations: (i) asaniczka's mid-senior native label is over-applied (arshkon seniority_native has a more balanced distribution including entry/associate); (ii) the LLM routes asaniczka rows to mid-senior by default when explicit signals are weak, reinforcing the asaniczka-native skew.
    • YOE-based asymmetry is smaller but still present: S4 arshkon 65% vs asaniczka 72% = 7.1 pp gap. S5 arshkon 24% vs asaniczka 28% = 4.0 pp gap. S4_rule arshkon 60% vs asaniczka 66% = 6.3 pp gap.

    Interpretation. The asymmetry does NOT disappear under the LLM YOE variant, but it shrinks by 60-70% (21 pp → 7 pp). This is consistent with real compositional differences between the two 2024 sources (asaniczka is a broader general-listings snapshot that includes more mid-to-senior ICs; arshkon focuses on entry-heavy LinkedIn employment postings) plus some residual label drift.

    Per preamble guidance: "If the asymmetry persists under the LLM-YOE variant, treat arshkon-only as a senior-side sensitivity (not a mandated primary)." The 7-pp S4 asymmetry persists. Therefore:

    • S4 primary claims should report the pooled-2024-vs-scraped effect (which is −7.6 pp) and note that this effect is roughly the same magnitude as the within-2024 arshkon-vs-asaniczka noise floor (−7.1 pp).
    • A senior-side sensitivity should report arshkon-only 2024 vs scraped 2026: S4 arshkon 65.0% → scraped 63.2% = −1.8 pp. This is close to the MDE of 2.5 pp for arshkon-vs-scraped and is not significantly non-zero under 80% power. The direction is still down but the magnitude collapses to within the noise floor.

    This is the single most important caveat downstream consumers must respect for senior-side claims. Junior-side is not affected — the J3 cross-period effect is +5.0 pp vs +4.8 pp within-2024 calibration, so both within-2024 sources agree on the direction relative to 2026 and the effect size exceeds the within-2024 noise.

    Junior side: 7/7 definitions consistent (all up). Clean direction verdict: junior share rose 2024→2026.

    Senior side: ⅚ definitions consistent (all down). S2 (director-only label) is flat (+0.32 pp), but this is within the MDE of 0.15 pp × ~2 and the director cell was flagged by T03 as low-precision (22-27% cell accuracy). S2 should not be weighted against the other 5 senior definitions; it is the noise floor, not a competing signal.

    Magnitude interpretation matters. Label-based senior definitions (S1, S3) show −15 pp drops; YOE-based S4 shows −7.6 pp; YOE-based S4_rule shows −7.6 pp; S5 (stricter YOE ≥ 8) shows −5.5 pp. The YOE-based primaries are about half the magnitude of the label-based. This is because:

    1. The unknown share grew from ~34% (asaniczka 2024) to ~52% (scraped 2026), mechanically reducing the share of rows labeled mid-senior under S1 without any change in underlying composition.
    2. Labeled senior positions in asaniczka 2024 are over-represented (64.2% under S1; asaniczka native labels skew to mid-senior by the companion dataset's label imbalance).

    Net verdict: the senior-share-falling story is real (all 5 primaries agree on direction) but the magnitude should be cited from the YOE primaries (S4), not the label primaries (S1). The S4 magnitude of ~7.6 pp is the most defensible point estimate.

    6.1 Title-keyword precision spot-check (J5/S3)¶

    J5 on scraped 2026 = 2.47% share (1,121 rows); J5 overlap with J1 (seniority_final='entry') is 65% (i.e., 65% of J5 rows in pooled-2026 are also entry by the label). The remaining 35% of J5 rows are title-keyword-junior but labeled unknown or mid-senior in seniority_final. Sampling 20 of those: they are primarily "Junior" titles that the LLM called unknown because the description didn't restate the level. The junior regex has no obvious false positives; the 2024 vs 2026 direction under J5 is stable.

    S3 on scraped 2026 = 41.89% (18,981 rows); S3 overlap with S1 is 92%. The 8% of S3 rows not in S1 are senior-title rows labeled as entry/associate/unknown by the LLM (rare). I did not find evidence of regex false positives driving the S3 number.

    No regex defect flagged. J5/S3 are valid sensitivities.

    7. Primary recommendation¶

    Adopt the preamble's defaults:

    • J3 (yoe_min_years_llm ≤ 2) as the junior primary.
    • S4 (yoe_min_years_llm ≥ 5) as the senior primary.

    Both meet the MDE bar (effect sizes 5-6× MDE) for pooled-2024-vs-scraped. No override needed for general claims.

    Mandatory caveats when citing primaries:

    1. Senior-side magnitude is contingent on 2024 source mix. Report S4 pooled-2024-vs-scraped (−7.6 pp) as the primary but always pair with S4 arshkon-only-vs-scraped (−1.8 pp) as the sensitivity. The arshkon-only estimate is not significant at 80% power (MDE 2.5 pp); the pooled estimate is.
    2. Junior-side magnitude is robust across 2024 sources. J3 within-2024 calibration (+4.8 pp arshkon−asaniczka) aligns with the cross-period effect direction; arshkon-only cross-period is +1.2 pp (13.0% → 14.2%), asaniczka-only cross-period is +5.9 pp (8.2% → 14.2%). Both are positive but the asaniczka-only effect dominates the pooled estimate because asaniczka is much larger. Junior claims citing pooled-2024 are defensible; arshkon-only is a weaker sensitivity.
    3. Always report the unknown share alongside label-based sensitivities. For J1/J2/S1, the unknown share shifted from 26% (arshkon) / 34% (asaniczka) to 52% (scraped), mechanically inflating label-based effect magnitudes. YOE-based primaries avoid this.
    4. Do not use S2 (director-only) as a primary. T03 established 22-27% cell accuracy on directors; T30 shows S2 cross-period is flat. Use S1 as the aggregate senior definition when a label-based measure is needed.

    Denominator convention for downstream tasks:

    • For label-based definitions (J1, J2, J6, S1, S2): report both share_of_all (denom = all SWE) and share_of_denominator (where denom = all SWE, which is the same). Also report the unknown share.
    • For YOE-based LLM definitions (J3, J4, S4, S5): report share_of_denominator (denom = yoe_min_years_llm IS NOT NULL AND llm_classification_coverage = 'labeled'). The panel includes both columns; consumers should standardize on share_of_denominator for YOE-based claims.
    • For YOE-based rule definitions (J3_rule, S4_rule): report share_of_denominator (denom = yoe_extracted IS NOT NULL).
    • For title_keyword definitions (J5, S3): share_of_all = share_of_denominator since the denominator is all SWE rows.

    8. Essential sensitivities¶

    8.1 Aggregator exclusion (a)¶

    Not recomputed in T30 directly — deferred to downstream task consumers. T03 established that seniority_final kappa is aggregator-insensitive in scraped (0.63 vs 0.66 excluding unknown), so the panel values here should not shift materially if the aggregator subset is excluded. Agent or wave-2 tasks that exclude aggregators can use the same panel definitions and will see slightly reduced n_of_denominator.

    8.2 Within-2024 calibration (f)¶

    Explicit in the panel's within_2024_effect column (arshkon − asaniczka for each definition). Summary:

    Definition within_2024_effect Direction of within-2024 noise Sign concordance with cross-period effect
    J1 +2.46 pp arshkon more entry concordant with up
    J3 +4.75 pp arshkon more entry concordant with up
    J4 +7.36 pp arshkon more entry concordant with up
    S1 −20.88 pp arshkon less senior (label) concordant with down
    S4 −7.09 pp arshkon less senior (YOE) concordant with down

    The within-2024 direction matches the cross-period direction for every primary. This means "arshkon is positioned like a junior 2026 sample relative to asaniczka 2024" — which is the expected pattern if the junior-expansion story is real and arshkon just happens to have a labor-market mix closer to the 2026 sample. It does NOT cleanly validate the cross-period effect (because arshkon is also 2024), but it at least doesn't contradict it.

    9. Surprises & unexpected patterns¶

    1. Every primary is monotonic in the predicted direction with no contrarians. I went into this expecting at least one label-based definition to flip (given the T03 abstention-and-director-weakness issues), but all junior definitions are up and all senior definitions except S2 are down. The signal is strong enough to be visible through any operationalization that isn't limited to the tiny director cell.
    2. Label-based magnitudes are ~2× YOE-based magnitudes on the senior side. S1 says −15.7 pp, S4 says −7.6 pp. This is not an error; it is driven by the unknown-share expansion from 26-34% (2024) to 52% (scraped). A naive reader would cite S1; the right number is S4.
    3. The junior effect under J3 (+5.0 pp) is much bigger than under J1 (+1.8 pp). The label-based J1 definition is conservative (only rows the LLM explicitly labels entry) while the YOE-based J3 captures the true junior-bar population more faithfully. J3 is the primary; J1 is the floor.
    4. J3_rule closely tracks J3 (cross-period +5.9 vs +5.0 pp, within-2024 +4.6 vs +4.8 pp). The rule extractor is a perfectly adequate audit for the junior-side YOE claim. This is somewhat reassuring given that an LLM might have drifted between periods; the rule is identical across periods by construction.
    5. S2 (director-only) is the single non-directional cell on the senior side. Two forces cancel: directors are a tiny cell (<1% of SWE everywhere), and the T03 audit showed 22-27% cell accuracy. The result is +0.32 pp which is within the MDE of 0.15 × 2 ≈ 0.3 pp. This is noise, not a contrarian signal.
    6. The S4 direction survives the asaniczka asymmetry but the magnitude does not. S4 arshkon-only-vs-scraped is −1.8 pp (within MDE of 2.5 pp); S4 pooled-2024-vs-scraped is −7.6 pp. The honest claim is "the senior share fell, and depending on which 2024 source you anchor on, the drop is between 2 and 8 percentage points." Downstream tasks must report both.
    7. LLM-vs-rule YOE agreement is 79-83% exact. The preamble threshold of 80% is met for arshkon and scraped; asaniczka is 79.2% — just below the flag line. The MAE of ~0.5-0.7 years suggests the disagreements are mostly within ±1 year. Not a red flag, but worth noting for any downstream task that needs fine-grained YOE bins.

    10. Data caveats¶

    • MDE calculations assume independent random samples. With clustering by employer (many postings per company), effective sample size is smaller and MDEs are optimistic. The within-2024 effect sizes already incorporate this noise floor empirically, so relative comparisons are robust; absolute MDE is a lower bound.
    • Scraped 2026 n is a snapshot. Re-running T30 after more scraped data accrues will tighten MDEs and may shift effect magnitudes slightly. The scraped 2026-03 and 2026-04 shares are stable so this is a low-risk issue for the current run.
    • selected_for_llm_frame is the basis for LLM-labeled rows. The YOE-based primaries are computed on the labeled frame only (see DENOM_PREDICATE). Out-of-frame rows are still available via the rule-based audit (J3_rule, S4_rule), which agrees closely.
    • Within-2024 pooled share uses the same data twice (asaniczka dominates the pooled estimator because it is 4× larger than arshkon). The arshkon-only 2024 baseline is the honest alternative; the panel reports both.

    11. Downstream action items¶

    1. All Wave 2+ seniority-stratified tasks should load exploration/artifacts/shared/seniority_definition_panel.csv — do not recompute these shares. The canonical schema is fixed.
    2. Primary junior/senior claims should cite J3/S4 from the panel. Label-based sensitivities (J1/S1) should be reported alongside with the unknown-share shift noted.
    3. Senior-side magnitude citation should always pair pooled-2024 with arshkon-only (S4 pooled = −7.6 pp; S4 arshkon-only = −1.8 pp). This is the cleanest way to expose the asaniczka asymmetry.
    4. Do not isolate directors (S2) as a primary finding. T03 and T30 both flag the director cell as unreliable.
    5.
