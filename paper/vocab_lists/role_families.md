## Labeling rules

- **Multi-label.** A posting can be assigned multiple families when warranted (e.g. "Senior Backend Engineer, ML Platform" → Backend + ML Engineer).
- **Software Engineer (General) is a residual fallback.** Apply only when no more specific family fits.
- **Full-stack tie-break.** Full-stack roles are tagged as Frontend/Web Engineer.

## Families

- **Software Engineer (General):** Residual fallback for SWE postings without a clear specialization. *Definition Source:* SFIA "Software engineering practitioners"【49†L281-L289】. *Examples:* Software Engineer, Software Developer (no further qualifier).

- **Frontend/Web Engineer:** Focused on user-facing web/mobile interfaces and UI/UX; full-stack roles default here. *Definition Source:* O*NET "Web Developers"【65†L67-L75】. *Examples:* Web Developer, Frontend Engineer, UI/UX Engineer, Full-Stack Engineer.

- **Backend/API/Services Engineer:** Focused on server-side logic, APIs, databases, and services. *Definition Source:* Industry practice; SFIA "Software engineering practitioners"【51†L75-L79】. *Examples:* Backend Engineer, API Developer, Microservices Engineer, Application Developer.

- **Mobile Engineer:** Specialist in native or cross-platform mobile app development. *Definition Source:* Industry practice (no single taxonomy). *Examples:* iOS Engineer, Android Developer, Mobile Application Engineer.

- **Embedded Engineer:** Builds firmware, IoT, and hardware-near software. *Definition Source:* SFIA "Real-time/embedded systems development"【49†L297-L304】. *Examples:* Embedded Systems Engineer, Firmware Developer, IoT Engineer.

- **Data Engineer / Data Platform:** Builds and maintains data pipelines, warehouses, and ingestion infrastructure. *Definition Source:* SFIA "Data engineering practitioners"【47†L220-L229】. *Examples:* Data Engineer, ETL Engineer, Data Platform Engineer.

- **Machine Learning (ML) Engineer:** Builds ML pipelines, trains models, and productionizes ML systems (pre-LLM). Distinguished from Data Scientist by focus on engineering ML systems rather than analytical results. *Definition Source:* SFIA "Data science practitioners" (ML skill set)【47†L190-L198】. *Examples:* ML Engineer, AI/ML Engineer (traditional), MLOps Engineer.

- **AI / LLM Engineer (Emergent):** Develops LLM-based or agent systems (prompt engineering, RAG, foundation-model integration). Newly emerging title category, tracked separately to measure its growth. *Definition Source:* Newly emerging in job postings (Indeed, LinkedIn trend reports); no formal taxonomy. *Examples:* AI Engineer, LLM Engineer, GenAI Engineer, Agent Engineer.

- **DevOps / SRE / Platform Engineer:** Ensures CI/CD, infrastructure-as-code, and reliability. *Definition Source:* SFIA "DevOps" role family【49†L354-L362】. *Examples:* DevOps Engineer, Site Reliability Engineer, Platform Engineer.

- **Security Engineer:** Focuses on application or infrastructure security. *Definition Source:* SFIA "Information and cyber security" role family【54†L496-L504】. *Examples:* Security Engineer, Application Security Engineer, Cybersecurity Engineer.

- **QA / Test Engineer:** Focuses on software quality, testing, and validation. *Definition Source:* SFIA "Testing" role family【54†L627-L635】. *Examples:* QA Engineer, Test Automation Engineer, QA/Test Lead.

- **Solutions / Field / Customer Engineer:** Deploys and integrates solutions with customers (pre-sales and post-sales). *Definition Source:* Industry practice; SFIA "Solution Architecture"【68†L13-L21】. *Examples:* Solutions Engineer, Field Applications Engineer, Customer Engineer, Pre-Sales Engineer.

- **Legacy Technology Specialist:** Maintains older or vendor-locked stacks in declining ecosystems. Non-exhaustive list — extend as new examples appear in the data. *Definition Source:* Industry practice; no formal taxonomy. *Examples:* Mainframe Developer, COBOL Engineer, Legacy .NET Developer, Salesforce Developer, ServiceNow Developer, SAP/ABAP Developer, Oracle Forms Developer.

- **Data / Analytics:** Transforms raw data into analytical, operational, or decision-support outputs; focused on delivering results and defining models rather than building production ML pipelines. *Definition Source:* SFIA "Data and analytics" role family ("Data science practitioners"【47†L190-L198】); O*NET "Data Scientists" 15-2051.00. *Examples:* Data Scientist, Data Analyst, Analytics Engineer, Data Visualization Developer.

- **Research Scientist / Engineer:** Conducts applied or basic research in ML, AI, robotics, or computational sciences; bridges academic research and product. *Definition Source:* Industry practice; partial overlap with SFIA "Data science practitioners"【47†L190-L198】. *Examples:* Research Scientist, Applied Scientist, Research Engineer.

- **Infrastructure Operations / Admin:** Operates and administrates databases, servers, and networks rather than authoring new software. *Definition Source:* O*NET "Network and Computer Systems Administrators" 15-1244.00; SFIA "Systems installation and removal" skill. *Examples:* Database Administrator (DBA), Systems Administrator, Network Engineer, Cloud Administrator.

- **People Manager:** Engineering management focused on people, hiring, and team operations. Excludes director/VP/C-level (out of scope for this taxonomy). *Definition Source:* SFIA "People management" skill family. *Examples:* Engineering Manager, Software Engineering Manager, Development Manager.
