    T08: Distribution Profiles

    Finding

    SWE job postings lengthened substantially: median description grew from 3,124 chars (2024-04) to 5,110 chars (2026-03), a 64% increase. The entry-level share (using seniority_patched) rose from 0.6% (2024-01, asaniczka) to 21.0% (2024-04, arshkon) to 13.9% (2026-03, scraped). However, the asaniczka 0.6% figure is misleading because asaniczka has no native entry-level labels -- its entry share is purely from title-keyword detection. Comparing arshkon to scraped (both with native labels), the entry share dropped from 21.0% to 13.9%. Meanwhile, remote work rose from effectively 0% to 24.7%, and aggregator rates dropped from 28.0% to 12-14%.

    Implication for analysis

    1. RQ1 (junior share): The arshkon-to-scraped comparison shows entry share declining from 21.0% to 13.9% (-7.1pp). This is the most apples-to-apples comparison since both sources have native entry-level labels. Asaniczka cannot contribute to entry-level trend analysis due to its labeling gap.
    2. Description length growth (64% median increase) means all text-based analyses must normalize by document length. Raw keyword counts would be biased upward for 2026-03.
    3. YOE drop (median 5 -> 3 years) in 2026-03 is directionally consistent with more junior postings, but coverage dropped from 35-40% to 16%, so this may be non-representative.
    4. Remote work comparisons across periods are not viable due to Kaggle data collection gaps.
    5. Aggregator rate difference (28% vs 12-14%) warrants sensitivity analysis with aggregators excluded.

    Data quality note

    1. Asaniczka's seniority distribution is an artifact: 100% native coverage but zero entry-level labels produces 0.56% entry share (purely from title keyword detection). Do not include asaniczka in entry-share trend analysis.
    2. YOE extraction coverage dropped substantially in 2026-03 (16.4% vs 34-40%). YOE-based analyses have selection bias.
    3. Remote flag is systematically missing for Kaggle sources -- treat the cross-period remote comparison as data artifact, not finding.
    4. The seniority patched workaround reduces unknown from 29-55% to 0-20%, but the asaniczka 0% unknown rate is because all rows get native backfill (all mid-senior/associate). This creates a mechanical dominance of mid-senior in asaniczka.

    Action items

    1. For RQ1 analyses: Use arshkon vs scraped as the primary comparison pair. Asaniczka provides volume context but not entry-level trend signal.
    2. Normalize text metrics: All keyword counts, n-grams, and text features must be normalized by description length to avoid systematic 2026-03 bias.
    3. Run aggregator-excluded sensitivity: The 28% vs 12-14% aggregator rate difference could shift seniority distributions.
    4. Remote analysis: Either drop cross-period remote comparisons or only report the 2026-03 level descriptively.
    5. YOE findings are suggestive but not primary: Low coverage in 2026-03 limits reliability. Use as supporting evidence, not standalone finding.
