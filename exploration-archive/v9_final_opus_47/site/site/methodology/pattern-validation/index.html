    Pattern validation¶

    Why this matters¶

    Longitudinal posting-content studies routinely claim "management language fell" or "AI tools rose" based on regex patterns whose semantic precision is never audited. V1 exposed that a widely-cited broad management pattern has 0.28 precision on a stratified 50-row sample — 72 of 100 matches are false positives (generic "team", "stakeholder", "coordinate" tokens in non-management contexts).

    The exploration phase adopted semantic-precision audits as a mandatory step before any pattern is used in a published claim.

    V1 + T22 validation protocol¶

    1. Stratified 50-row sample (pre-period + post-period) is drawn from the full corpus for each pattern.
    2. Each match is manually labeled true positive (TP) or false positive (FP) based on whether the matched span is semantically consistent with the pattern's intended meaning.
    3. Thresholds:
    4. Precision >= 0.80 → PRIMARY
    5. 0.60 <= precision < 0.80 → DIAGNOSTIC
    6. Precision < 0.60 → FAIL
    7. Per-sub-pattern precisions are reported (hire: 0.07, performance_review: 0.25, etc.), not just the aggregate.
    8. By-period precision is reported to detect period-specific contamination (e.g., "MCP" = Microsoft Certified Professional in 2024, not Model Context Protocol).

    Key failure modes¶

    • mgmt_broad 0.28: "team", "stakeholder", "coordinate" match in generic collaboration contexts, not management responsibility.
    • mgmt_strict hire 0.07: overwhelmingly "contract-to-hire", "direct-hire", "how-we-hire/accommodations" — HR metadata, not managing hiring.
    • mgmt_strict performance_review 0.25: primary FP is "code review", "peer review" in QA contexts.
    • ai_broad mcp_2024 0.15: "MCP" = Microsoft Certified Professional in 2024 (pre-MCP-protocol era). Direct within-2024 contamination.

    Principle: patterns need semantic, not tautological, precision¶

    A pattern that matches tokens related to management can have high token-level precision (mentions what it says it mentions) while having low semantic precision (those mentions are not actually management responsibility). The distinction is between word presence and meaning fidelity.

    The exploration's methods contribution is partly this protocol: cite semantic precision per pattern, per period, per sub-pattern, with a stratified 50-row manual audit. Every downstream claim should name the pattern and its precision.
