    T36 — Legacy-stack substitution map (H_L)¶

    Agent S · Wave 3.5 · 2026-04-20 Input: data/unified.parquet (filter frame) + swe_cleaned_text.parquet (text_source='llm' only) + T10 disappearing-titles lists (disappearing_titles.csv strict n=2; disappearing_titles_arshkon_ge5.csv wider n=11) + V1-validated ai_strict pattern for AI-vocabulary comparison. Method: TF-IDF centroid (1-2 grams, 40k features, sublinear TF, NLTK + company stoplist, min_df=5) per title on cleaned descriptions; cosine between 2024 disappearing-title centroid and 2026 title-universe centroids (N ≥ 10 postings per 2026 title). Company cap 50 per title-period. Fightin' Words (Monroe/Colaresi/Quinn log-odds with informative Dirichlet prior, α=0.01) for content drift per top-1 pair. Script: exploration/scripts/T36_legacy_stack_substitution.py (runtime 5.0 s). Tables in exploration/tables/T36/.

    Scope caveat (as flagged in dispatch)¶

    H_L has modest evidence. n=6 titles with reasonable sample sizes. Report as exploratory substitution map rather than decisive finding.

    • Arriving: environments (-2.9), "cd pipelines" (-2.1), workflows (-1.8), manage (-1.7, mgmt), tooling (-1.4)

    Clean read: architect + bash/unix scripting + management language → CD pipelines + workflows + tooling. Modernization within-role.

    drupal developer → web developer (cosine 0.336)¶

    • Leaving: drupal (7.8, legacy_tech), modules (4.9), months (4.4), php (4.4, legacy_tech), scrum (3.9, methodology)
    • Arriving: website (-2.2), seo (-2.1), digital (-1.9), optimization (-1.8), ui (-1.7)

    Clean read: drupal/php CMS engineering → website/SEO/digital/UI marketing-focused web work. Tech specialization dropped; role broadens.

    java architect → java developer (cosine 0.532)¶

    • Arriving: maintain (-3.0), backend (-2.9), "java boot" (-2.7), pipelines (-2.2), "cd" (-2.1), "boot microservices" (-2.1)

    Clean read: architectural/design language + consulting ("clients", "verbal", "written" — communication with client stakeholders) → Spring Boot / microservices / CD. Individual-contributor spring-boot engineering replaces architect-consulting.

    java application developer → full stack java developer (cosine 0.457)¶

    scala developer → big data engineer (cosine 0.297)¶

    • Arriving: pipelines (-2.1), solutions (-1.6), etl (-1.5), "cd" (-1.0), continuous (-0.9)

    Clean read: Scala-specific work (functional, batch, kinesis streaming) → generic ETL pipelines. Scala as a language dissolves; the work gets generic-ETL framing.

    Semantic-category tally across all 6 pairs (word-weight): - Leaving disappearing corpora: scope ("architect", "architectural", "designing"), mgmt ("management"), legacy_tech (drupal, php, ssis, scala). - Arriving 2026 corpora: modern_tech (postgresql, azure, microservices), methodology/CI-CD vocabulary (pipelines, cd, workflows, environments), generalist verbs (maintain, manage, tooling).

    No AI-tool terms rank among top 10-20 for any of the 6 pairs. The substitution is legacy-stack-to-modern-stack consolidation — not legacy-to-AI at this evidence threshold.

    devops architect devops engineer 0.00 0.020 +0.020
    drupal developer web developer 0.00 0.044 +0.044
    java application developer full stack java developer 0.00 0.024 +0.024
    java architect java developer 0.00 0.010 +0.010
    scala developer big data engineer 0.00 0.071 +0.071

    Disappearing-title 2024 AI-strict rate = 0.00 for all 6 titles — legacy stacks had no AI-strict tokens. 2026 neighbor AI-strict rate is 1-7% (mean 3.6%) — well below the aggregate 2026 SWE AI-strict rate of ~14.4% (calibration_table.csv).

    Interpretation: the 2026 titles that absorb disappearing-title work are below-average AI-strict penetration. Database/devops/web engineering isn't the part of the market where AI tokens concentrate (that's ai_ml archetype per T28 and T35 comm 2). The substitution is mostly legacy-tech-modernization (SSIS → pgvector + pipelines, unix scripts → CD workflows) rather than legacy-tech → AI-enabled replacement.

    Conclusion for H_L as framed: "legacy consolidation → AI-enabled roles" interpretation is NOT strongly supported in the T36 substitution map. The 2026 neighbors modernize the stack (modern DBs, CI/CD, microservices) but do NOT systematically increase AI-strict requirements beyond background.

    But: Δ is positive for all 6 pairs (+0.01 to +0.07) — AI mentions are somewhat higher in the 2026 neighbor than in the 2024 disappearing title. The weaker claim "disappearing legacy titles absorb into 2026 roles with modestly higher AI-mention rates" is supported; the stronger "AI-enabled roles replace legacy titles" is not.

    Step 4 — manual inspection (10 postings per pair side, truncated to 600 chars)¶

    Read samples side-by-side from manual_inspection.csv. Pattern-level assessment per pair:

    • database developer → database engineer: CREDIBLE. 2024 postings (tCognition, Kavaliro, Ria Money Transfer) emphasize relational SQL + NoSQL + ETL + designing cloud-native data architectures. 2026 postings (Echelon Services — TS/SCI clearance, Peraton — govt-contractor) emphasize Postgres+pgvector / neo4j / elasticsearch + schema tuning + backup/replication. Same work-shape with a modern tool set. Substitution genuine.

    • devops architect → devops engineer: CREDIBLE. 2024 postings (Ascentt, Quantum World, Sapphire Software, Zodiac Solutions) are consultant-contracting roles emphasizing "architect" + Terraform + CI/CD + K8s management. 2026 postings (OpenGov — SaaS, Mach Industries — defense, Salesforce-like) are in-house platform roles emphasizing GitHub Actions + Terraform + K8s + Azure/AWS + ArgoCD + Grafana + observability. Same stack; the 2024→2026 title change reflects staffing mode (contractor-architect → in-house-engineer) as much as content.

    • java architect → java developer: CREDIBLE. 2024 postings (Quantum World, Digitive, Fixity — consultant staffing) emphasize Java architect + Spring Boot + microservices + Kafka + architectural principles + client communication. 2026 postings (in-house Java engineer roles) emphasize Spring Boot + microservices + CD. Lose the architect/client-facing/communication language; keep the Spring stack. Staffing-mode shift visible.

    • drupal developer → web developer: CREDIBLE with noise. 2024 postings (Apetan Consulting, SKILL BIRD, Esteemed, Aimic) are clearly contract-staffing Drupal CMS roles (remote/hybrid, months-long, php/javascript/html/css). 2026 web-developer postings include SEO/marketing-adjacent web work (cosine 0.34 reflects partial overlap only). Genuine substitution into a lower-specialization bucket; some loss of technical specificity.

    • java application developer → full stack java developer: CREDIBLE. 2024 postings emphasize backend Java applications + systems. 2026 postings emphasize Java + Angular + Spring Boot + microservices + CI/CD. The 2026 role is strictly broader.

    • scala developer → big data engineer: PARTIALLY CREDIBLE. 2024 postings (CodeForce 360, Dexian, Empower) are Scala+Hadoop+Spark+Kafka+Storm+HDFS data-infra roles. 2026 "big data engineer" (n=14, small sample) includes a cluster of BAE Systems defense-contractor ETL roles that are thinly described ("architectures etl oversees implementation solutions"). Substitution is less clean here (cosine 0.30 is weakest); the thin 2026 match is partly driven by the small n=14 of the 2026 title rather than deep content alignment.

    All 6 substitutions are credible at the "same work, different title+tools" level. None are loose cosine matches where the content mapping fails on read.

    Surprises¶

    1. Architect titles are disappearing, not just legacy-tech titles. Both devops architect and java architect are in the wider list. The "architect" job-title is consolidating into "engineer" — the title-vocabulary is compressing. This is broader than a legacy-tech substitution story; it's a title-inflation reversal where the 2024 architect-consulting staffing pattern gives way to in-house engineer hiring. Consistent with T10's "-12 pp senior title-marker share drop, +2.7 pp staff doubling" finding.

    2. No disappearing title maps UPWARD. All 6 substitutions are same-level or downward in seniority. Given the 2024 architect titles had YOE 9-10 vs 2026 developer/engineer YOE 5-6, the substitution involves a 2-4 year YOE drop on average. This aligns with Gate 2's "senior title-marker dropped 12pp" finding and suggests the 2024 architect-consultant staffing pattern (which inflated YOE in the arshkon/asaniczka panel) is not reproducing in 2026 scraped.

    3. AI mentions do NOT replace legacy stacks in this substitution map. 2026 neighbors sit at 1-7% AI-strict rate, below the market-wide 14.4%. The "legacy SWE → AI-enabled SWE" story does NOT hold at the substitution-pair level. What we see instead is legacy stack → modern stack with CI/CD + observability + microservices + modern DBs. Gate 2's AI-rewriting story is a separate phenomenon concentrated in ai_ml archetype postings (per T35 comm 2), not in the legacy-stack substitution channel.

    4. Aggregator staffing is the main source of disappearing titles. Inspection of 2024 top employers in disappearing_titles_arshkon_ge5.csv shows aggregators/staffing firms (Infosys, Photon, Quantum World Technologies, Fixity, Zodiac Solutions, Apetan Consulting, CodeForce 360, Dexian, TEKsystems) dominate the 2024 side. These firms are less prominent in 2026 scraped (partly because asaniczka had a higher aggregator share than scraped — see Gate 1). Whether disappearance reflects a genuine market shift or a change in the scraping/sourcing mix is a sampling-frame question (T37).

    5. "Scala developer" pair is the weakest match (cosine 0.297). This is expected — scala as a technology declined in phi rank but didn't fully disappear from 2026, so the nearest generic title is a weak match. Scala developers probably consolidated into multiple 2026 titles (big data engineer, data engineer, software engineer) rather than a single successor.

    6. T10's title-normalization differs from unified's. 5 of 11 wider-list titles fail to match in the T36 join. This is a title-normalization mismatch (T10 applies its own regex; unified uses the pipeline's title_normalized). For the strict list (n=2), 1 of 2 survives (java architect only; senior dotnet developer doesn't match). This warrants a note in the Wave 4 synthesis: T10's "disappearing" designation is slightly tighter than what downstream cross-frame matching can reproduce.

    Limitations¶

    1. n=6 disappearing titles. H_L is power-limited. Findings should be read as case-study-plus-aggregate rather than population-level.

    2. TF-IDF centroid is mean-pooled on cleaned descriptions. If a disappearing title has 8 postings and some are short or boilerplate-heavy, the centroid can drift. Company-cap 50 caps boilerplate dominance but doesn't eliminate within-corpus variance.

    3. 2026 title universe is 174 titles at N ≥ 10 (universe_titles in run_meta.json). Smaller-volume 2026 titles (N < 10) are excluded; a disappearing title's "true" nearest neighbor in the modern market could be a lower-volume specialty title that we don't see.

    4. Seniority ordinal conversion (entry=1, associate=2, mid-senior=3, director=4, executive=5) is coarse. "Same level" classifications are within 0.3 of this ordinal, ~30% of one seniority bin. YOE is a cleaner proxy but has 30-50% missingness within disappearing-title corpora.

    5. ai_strict precision is 0.86 per V1. The 0.00 AI-strict rate on disappearing-title corpora is likely clean; the 1-7% AI-strict rate on 2026 neighbors could include some FPs (e.g., "agent" false positives, "fine-tuning" non-AI uses). Directional interpretation ("2026 neighbors have low AI rates") survives this precision slop.

    6. Fightin' Words Z-scores are relative to the two-corpus background only. A word like "architect" scoring z=8.1 favoring disappearing reflects "within this pair, architect is heavily concentrated in the 2024 corpus" — not "architect is a 2024-specific word in the full 2024 market." Interpret qualitatively.

    Headline claims for SYNTHESIS¶

    • 6 of 11 widened-list disappearing 2024 titles find a credible 2026 neighbor via TF-IDF centroid cosine ∈ [0.30, 0.59]; top-1 neighbors are all "engineer" or "developer" variants absorbing "architect"/"application"/"legacy-tech" prefixes. — evidence: substitution_table_top1.csv, manual_inspection.csv, sensitivity verdict: partial (n=6 titles, mid-cosine; aggregator-excluded preserves ordering but shows cosine drops 0.02-0.08).

    • Architect titles (devops architect, java architect) substitute into engineer titles with a 2-4 year mean-YOE drop (9.4→5.3; 9.9→6.3) and loss of architect/mgmt/scope language; this is title-inflation reversal not tech-modernization. — evidence: substitution_table.csv (rank=1 rows), content_drift_per_pair.csv (leaving: architect/architectural/designing/clients), sensitivity verdict: partial (2-of-11 evidence, but content drift direction is clean).

    • Disappearing-title substitution is legacy-stack-to-modern-stack, NOT legacy-stack-to-AI-enabled. 2026 neighbor corpora run at 1-7% ai_strict (mean 3.6%), below the 14.4% 2026 market-wide rate. The "legacy-consolidation → AI-enabled roles" narrative is NOT supported in this substitution channel. — evidence: ai_vocab_comparison.csv, sensitivity verdict: robust (6/6 pairs agree; V1-validated ai_strict pattern).

    • Content drift pattern across the 6 pairs is uniform: SSIS/Unix-scripts/Drupal/PHP/Scala leave; Postgres/pgvector/CI-CD/CD-pipelines/microservices/ArgoCD/Grafana/Azure arrive. — evidence: content_drift_per_pair.csv (6 pairs), sensitivity verdict: robust (Fightin' Words Z-scores significant at p<0.05 for top-10 terms in all 6 pairs).

    • Substitution map is dominated by title-consolidation pattern (d): specialization-adjective-prefixed 2024 titles consolidate into the adjectiveless 2026 title; 6/6 pairs fit this. — evidence: substitution_table_top1.csv, sensitivity verdict: robust.

    • No disappearing title maps UPWARD; 4/6 are same-level, 2/6 downward — H_L's "upward consolidation" variant is unsupported. — evidence: substitution_table_top1.csv (seniority_shift column), sensitivity verdict: robust.

    • Strong caveat: T10 wider list is n=11; T36 can only match 6 via default filter title-normalization. The strict n=2 list yields only 1 matched case (java architect). H_L evidence is modest. — evidence: T36 scope caveat + manual inspection, sensitivity verdict: flagged (power-limited).
