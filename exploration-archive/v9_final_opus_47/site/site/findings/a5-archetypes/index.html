    A5 — Two emergent senior archetypes with content-grounded names¶

    Claim

    The senior SWE cohort (n=33,693) separates into two growing archetypes. Cluster 0 — Senior Applied-AI / LLM Engineer grew 15.6x from 144 to 2,251 postings; 94% of members are 2026. Cluster 1 — Senior Data-Platform / DevOps Engineer grew 2.6x from 2,034 to 5,258, capturing +11.4 pp of senior share. The AI-oriented cluster asks one year more experience (median YOE 6 vs 5) than the traditional senior role, contradicting the "AI lowers the bar" prior.

    Key figure¶

    Title distribution in cluster 0 vs cluster 1. Cluster 0 is 28% "AI Engineer" at face value (true share >= 45% per manual check), 18% senior engineer, 8% staff engineer, 6% tech lead. Cluster 1 is 20% senior engineer, 13% data engineer, 11% DevOps/SRE.

    Cluster profiles¶

    Cluster 0 — Senior Applied-AI / LLM Engineer¶

    • Growth: 15.6x (144 → 2,251), 94% 2026.
    • Titles: 28% "AI Engineer" at face value; >= 45% true share per manual check.
    • T09 archetype cross-tab: models/systems/llm 6.75x over-rep, systems/agent/workflows 5.65x over-rep.
    • Distinguishing bigrams (log-odds z, per T34): claude code (+5.52), rag pipelines (+5.04), github copilot claude (+5.02), langchain llamaindex (+4.86), augmented generation rag (+4.72), guardrails model.
    • Median YOE: 6.0 (vs 5.0 in cluster 1).
    • Director share: 1.9% (2x cluster 1).
    • Industry mix: Software Development 44.6% + Financial Services 16.5% + IT Services 13.6%.
    • Firm dispersion: HHI 38.6 across 1,163 distinct firms (genuinely distributed, not concentration-driven).
    • Recurring asks (T34 20 exemplars): LLM/RAG/prompt 12/20; pipelines 13/20; agentic 8/20; orchestration 7/20; system design 7/20.

    Cluster 1 — Senior Data-Platform / DevOps Engineer¶

    • Growth: 2.6x (2,034 → 5,258).
    • Titles: 20% senior engineer + 13% data engineer + 11% DevOps/SRE.
    • T09 archetype cross-tab: pipelines/sql/etl 2.62x over-rep, kubernetes/terraform/cicd 2.11x over-rep.
    • orch_density: 2.27 per 1K chars (highest across the 5-cluster partition).
    • Industry mix: Software Development 43.1% + IT Services 14.5% + Financial Services 11.9%.
    • Distinguishing bigrams: training benchmarking, annotation validation, dask spark, benchmarking pipelines, automation-first.
    • Note: visibly a bundle of data-engineering + DevOps/SRE + AI-lab data-contract work. A k=6 or k=7 clustering would likely split this further.

    Sensitivity verdict¶

    Strong for cluster 0, moderate for cluster 1.

    • V2 Phase B: 14 of 20 sampled cluster-0 titles are explicit AI/ML.
    • V2 Phase C: cluster 0 share is robust across S1/S2/S3/S4 T30 panel variants — 10 to 14x rise under each.
    • Both clusters robust to aggregator exclusion (<3% metric shift).
    • Cluster 1 heterogeneity: analysis phase should pre-register k via BIC or gap-statistic; expect the bundle to split.

    Falsifies / refines priors¶

    • "AI lowers the bar" prior — contradicted. AI-senior roles ask +1 YOE more than traditional senior.
    • One-archetype-emerges prior (Gate 2) — refined. Two clusters emerged, not one.

    Dig deeper¶

    • Raw task reports: T34 (cluster profiling), T21 (k=5 k-means partition, silhouette 0.477).
    • Interview artifact: T25 artifact 3 (Applied-AI/LLM Engineer archetype exemplars — 20 posting excerpts).
